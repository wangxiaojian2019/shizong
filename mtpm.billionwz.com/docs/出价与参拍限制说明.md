# 出价与参拍限制说明

本文档根据当前代码梳理 **限时拍**、**同步拍** 的手动出价、代理出价及参拍资格限制。涉及主要文件：

| 模块 | 路径 |
|------|------|
| 出价 API | `app/api/controller/v1/store/StoreBiddingController.php` |
| 手动/代理出价逻辑 | `app/services/product/product/StoreBiddingServices.php` |
| 代理出价 CRUD | `app/services/product/product/StoreAgentPriceServices.php` |
| 号牌与参拍资格 | `app/services/product/product/StoreNumberPlateServices.php` |
| 限时拍保证金 | `app/services/product/product/StoreTimedDepositServices.php` |
| 限时拍小程序页 | `mtpm/pages/goods_cate/goods_cate2.vue` |
| 同步拍小程序页 | `mtpm/pages/goods_cate/synchronous_bidding.vue` |
| 出价失败 UI | `mtpm/utils/bidResult.js` |

---

## 1. 接口一览

| 类型 | 方法 | 路由 | 说明 |
|------|------|------|------|
| 限时拍手动出价 | GET | `auction` | `product_id`, `plate`, `price`（**整型**，单位：元） |
| 同步拍手动出价 | GET | `synchronousAddPrice` | 参数同上 |
| 设置代理出价 | GET | `agentPrice/add` | `product_id`, `agent_price`, `plate`, `model`（0=限时拍，1=同步拍） |
| 修改代理出价 | GET | `agentPrice/updateAgent` | `product_id`, `agent_price` |
| 取消代理出价 | GET | `agentPrice/cancelAgent` | `product_id` |

控制器对 `auction` / `synchronousAddPrice` 在失败时返回 `status: 400` + `msg` 文案；成功为 `status: 200` + `msg: 出价成功`。

**重试：** 控制器内 `while ($msg == '' && $sum < 3)` 最多调用服务 3 次；当前 `addPrice` / `synchronousAddPrice` **始终返回非空字符串**，该重试仅在为空的异常情况下生效，正常业务不重复执行。

## 2. 参拍资格（号牌 / 保证金）

所有 **手动出价**、**设置/修改代理出价**、**系统自动代理跟价** 均先走 `StoreNumberPlateServices::validateBidEligibility()`。

### 2.1 通用

| 条件 | 失败提示 |
|------|----------|
| `uid <= 0` 或 `cate_id <= 0` | 参数错误 |
| 保证金退款审核中（`application === 1`） | 保证金退款审核中，暂不可参拍 |
| 服务端解析不到有效号牌 | 号牌无效，请刷新专场后重试 |
| 客户端提交的 `plate` 与服务端号牌不一致 | 号牌状态已变更，请刷新后重试 |

服务端以数据库实时号牌为准，会覆盖请求中的 `plate`。

### 2.2 限时拍（`isTimedAuction(cate_id) === true`）

| 条件 | 失败提示 |
|------|----------|
| 未办理号牌（`plate_registered === false`） | 请先办理号牌 |
| 已办牌但未分配号牌（`has_plate === false`） | 号牌待分配，请联系客服分配或缴纳保证金 **{金额}** 元后系统自动分配 |

- 保证金金额：`StoreTimedDepositServices::getDepositAmount()`，配置项 `timed_auction_deposit_amount`，默认 **1 元**。
- 限时拍方案：**有号牌即可参拍**（`state = 1` 当 `has_plate`），不再因「保证金已退」单独拦截出价（见 `buildParticipationDetails` 注释）。

### 2.3 同步拍（非限时拍专场）

| 条件 | 失败提示 |
|------|----------|
| 无号牌（`has_plate === false`） | 请先办理号牌 |
| 保证金未生效（`state !== 1`） | 请缴纳保证金后再出价 |

同步拍另依赖号牌表字段 **`remaining`（剩余可竞拍额度）**，见第 5 节。

## 3. 拍品状态限制（手动出价）

逻辑在 `addPrice()` / `synchronousAddPrice()`，依据商品字段：

| 字段 | 含义（业务） | 限制 |
|------|----------------|------|
| `presale_end_time > 0` | 已结拍 | 返回 **已结拍**，不可出价 |
| `presale_start_time == 0` | 拍卖未开始 | 返回 **拍卖未开始** |
| `presale_start_time > 0` 且未结拍 | 拍卖进行中 | 可进入出价逻辑 |

### 3.1 出价金额与最高价

进行中时：

1. **`price`（出价）必须严格大于当前价 `productInfo['price']`**
   - `price <= productInfo['price']` → **出价失败,您的竞价已被超越**
   - 后端 **不校验** 加价阶梯是否为 500/1000，只校验「高于当前价」。

2. **若当前最高价已是本人**（`productInfo['uid'] == 出价用户 uid`）
   - 返回 **无需出价,您是当前最高**
   - 不更新 `version`，不写库。

3. **乐观锁更新**
   - `UPDATE` 条件：`id = product_id AND version = 当前 version`
   - 更新：`price`、`uid`、`version + 1`、`pattern = 1`（手动出价）
   - 影响行数为 0 → **出价失败**（并发被他人抢先）

### 3.2 落槌延时（限时拍）

手动出价成功且 **`productInfo['price'] <= productInfo['cost']`**（达到/低于保留价相关逻辑）时，专场 `delayCycle` 会延长 **120 秒**（从专场 `startTime` 起算剩余时间写入 `delayCycle`）。

## 4. 加价阶梯规则

### 4.1 限时拍 — 小程序前端（`goods_cate2.vue`）

前端在提交前做阶梯校正；后端仅校验「大于当前价」。

| 当前价区间 | 加价按钮步长 | 相对起拍价 `ot_price` 的网格 |
|------------|--------------|------------------------------|
| `< 50000` | ±500 | 超出部分按 **500** 对齐 |
| `>= 50000` | ±1000 | 超出部分按 **1000** 对齐 |

- 手动出价低于 `detail.price + 500`（或高价段规则）时，会自动抬到最低可出价并二次确认。
- 代理出价、手动出价共用类似阶梯算法（`getagent` / `getprice`）。

### 4.2 限时拍 — 系统代理自动跟价（`automatic()`，`model = 0`）

| 当前价 | 系统自动出价额 |
|--------|----------------|
| `price == 0` 且 `ot_price != 0` | `ot_price` |
| `price < 50000` | `price + 500` |
| 否则 | `price + 1000` |

- 若计算价 **大于** 用户设置的 `agent_price` → 代理标记为失效（`status = 2`），并通知用户。
- 否则在乐观锁成功时以 **`pattern = 0`**（代理出价）写入。

### 4.3 同步拍 — 小程序前端

- 出价前：`value > remaining` → **您的竞价额度不足**（仅前端拦截，后端 `synchronousAddPrice` 未再校验 `remaining`）。

### 4.4 同步拍 — 系统代理自动跟价（`synchronousAutomatic()`，`model = 1`）

与限时拍 **不同**，按价位与专场 `bidding_ladder` 计算：

| 当前价 | 加价规则 |
|--------|----------|
| `price == 0` 且有 `ot_price` | 使用 `ot_price` |
| `price < 100000` | 看 `(price/1000)%10`：为 0 或 8 则 **+2000**，否则 **+3000** |
| `price >= 100000` | `bidding_ladder == 1`：千位规则 +2000/+3000；否则万位规则 **+20000/+30000** |

## 5. 同步拍「竞拍额度」`remaining`

仅同步拍号牌有 **`remaining` / `freeze`** 等额度字段。

| 场景 | 行为 |
|------|------|
| 设置代理出价（`model = 1`） | `remaining -= agent_price` |
| 手动出价成功 | 新出价者：`remaining -= 成交价`；被超越者：退回其出价占用额度（`remaining += 被超越价`） |
| 代理失效/结拍 | 按 `synchronousAutomatic` 逻辑回退额度 |

限时拍手动出价 **不扣减** `remaining`。

---

## 6. 代理出价限制

入口：`StoreAgentPriceServices::saveData()` / `updateAgents()`。

| 限制 | 说明 |
|------|------|
| 参拍资格 | 同第 2 节 `validateBidEligibility` |
| 重复设置 | 已存在 `status = 1` 或 `2` 的记录 → **请不要重复设置代理出价** |
| 同步拍额度 | `model = 1` 时扣减 `remaining`，号牌无效 → **号牌信息无效，请刷新专场后重试** |
| 修改/取消 | 更新时同样校验资格；记录不存在 → **代理出价记录不存在** |

代理跟价由定时任务或出价成功后触发 `automatic()` / `synchronousAutomatic()` 执行，规则见第 4.2 / 4.4。

## 7. 返回消息对照表（手动出价）

| `msg` | 触发条件 | 建议用户操作 |
|-------|----------|--------------|
| 出价成功 | 乐观锁更新成功 | 刷新当前价与出价记录 |
| 出价失败,您的竞价已被超越 | 出价 ≤ 当前价 | 刷新后按新价加价 |
| 出价失败 | 乐观锁失败 / 重试后仍无文案 | 刷新后重试 |
| 无需出价,您是当前最高 | 当前价已是本人 | 无需再出 |
| 拍卖未开始 | `presale_start_time == 0` | 等待开拍 |
| 已结拍 | `presale_end_time > 0` | 查看下一件或订单 |
| 拍品不存在 | 商品 ID 无效 | 返回列表 |
| 请先办理号牌 | 限时/同步未办牌 | 办理号牌 |
| 请缴纳保证金后再出价 | 同步拍 `state !== 1` | 缴纳保证金 |
| 号牌待分配，… | 限时拍未分配号牌 | 缴保证金或联系客服 |
| 号牌状态已变更，请刷新后重试 | 客户端号牌过期 | 刷新专场 |
| 号牌无效，请刷新专场后重试 | 服务端无号牌 | 刷新专场 |
| 保证金退款审核中，暂不可参拍 | `application === 1` | 等待审核结束 |

小程序统一处理：`mtpm/utils/bidResult.js`（`handleTimedManualBidResult` / `handleSyncManualBidResult`）。

## 8. 限时拍 vs 同步拍差异摘要

| 项目 | 限时拍 | 同步拍 |
|------|--------|--------|
| 手动出价 API | `auction` | `synchronousAddPrice` |
| 参拍保证金 | 办牌 + 分配号牌（配置金额） | 须 `state=1` 缴纳保证金 |
| 额度 `remaining` | 不使用 | 使用，前端校验出价 ≤ remaining |
| 手动加价阶梯（前端） | 500 / 1000 | 页面自有规则 + 额度 |
| 代理 `model` | `0` | `1` |
| 自动跟价阶梯 | +500 / +1000 | +2k/3k 或 +2w/3w 等 |
| 出价成功后续 | `automatic()` 限时代理 | `synchronousAutomatic()` + 额度变动 |

## 9. 实现注意事项（开发）

1. **后端不校验加价阶梯**：若绕过小程序直接调 API，只要 `price > 当前价` 且资格通过即可成交。
2. **`price` 为整型**：小数会被截断。
3. **并发**：高并发下依赖 `version` 乐观锁，失败应提示刷新而非重复加价。
4. **缓存**：出价成功会更新 `product_{id}`、`bidding_{id}` 等缓存；资格变更后需刷新号牌接口。
5. **登录**：接口从 `$request->uid()` 取用户；`uid=0` 会在资格层返回 **参数错误**。
