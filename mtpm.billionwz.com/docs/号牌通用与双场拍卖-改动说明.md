# 号牌通用与双场拍卖 · 改动说明

> 需求依据：`docs/号牌通用与双场拍卖-最终方案需求.md`  
> 数据库脚本：`docs/database/auction_session_migration.sql`

## 1. 背景

限时拍（`model=0`）需支持**同一拍卖周期内最多 2 个专场并行**，号牌在周期内通用，保证金固定 **5000 元**且与号牌独立；同时保持**无 `session_id` 的单场现网逻辑**不变，同步拍（`model=1`）不做改动。

## 2. 数据库变更（需先执行迁移）

| 对象 | 说明 |
|------|------|
| `eb_store_auction_session` | 拍卖周期（前后两天） |
| `eb_store_timed_deposit` | 用户级限时拍保证金（`paid_total` / `paystatus` / `application`） |
| `eb_store_category.session_id` | 0=单场模式 A；>0=周期内双场模式 B |
| `eb_store_number_plate.session_id` | 0=按 `cate_id`；>0=按周期共享号牌 |

## 4. 业务规则实现

### 4.1 号牌模式

| 条件 | 模式 | 号牌维度 |
|------|------|----------|
| `category.session_id = 0` | A（现网） | 1 个 `cate_id` = 1 个号牌 |
| `category.session_id > 0` | B（新） | 1 个周期 = 1 个号牌，两专场共用 |

### 4.2 参拍校验

```
has_plate AND paid_total >= 5000 AND paystatus == 1
```

- `StoreNumberPlateServices::checkCanParticipate($uid, $cateId)`
- 用于：`StoreBiddingServices::addPrice`、`StoreAgentPriceServices::saveData`（仅 `model=0`）

### 4.3 保证金

- 用户级账户：`eb_store_timed_deposit`，与号牌表解耦
- 支付：仅 5000；出价**不扣减**保证金
- 周期结束**不自动作废**；用户退款后须重新缴纳
- 回调：`NotifyController` → `StoreTimedDepositServices::onPaymentSuccess`
- 退款申请：`StoreNumberPlateController::refund` → `markApplicationPending`

### 4.4 API `details` 响应字段

| 字段 | 说明 |
|------|------|
| `number_plate` / `has_plate` | 号牌 |
| `plate_mode` | `cate` / `session` |
| `paid_total` / `deposit_sufficient` / `need_pay` | 保证金 |
| `state` | **兼容旧前端**：1=保证金已缴足 |

## 5. 行为变化对照

| 项目 | 改前 | 改后 |
|------|------|------|
| 限时拍专场列表 | 最多 1 个 | 最多 2 个（同周期） |
| 保证金存储 | 号牌表 `state/paystatus` | 用户表 `eb_store_timed_deposit` |
| 5000 支付 | 写号牌表 | 写保证金表（号牌表仅过渡同步 `state`） |
| 出价/代理 | 无统一校验 | 须办牌 + 保证金足额 |
| 专场推进 cron | 仅 `category[0]` | 遍历缓存中各专场，独立 `productList_{cateId}` |
| 同步拍 | 1:5 比例 + `remaining` | **不变** |

## 6. 部署与配置

1. **执行 SQL 迁移**（见第 2 节）。
2. **清缓存**：部署后建议清除 `category`、`timed_deposit_*`、`Plates_*`、`Plate_*` 等相关键。
3. **绑定周期**：在后台将两个限时拍专场的 `session_id` 设为同一周期 ID（管理端 UI 待 P4 补充时可先 SQL 更新）。
4. **现网单场**：保持 `session_id=0` 即可，逻辑与改前一致。

## 7.1 业务逻辑审查修复（2026-05-29）

| 问题 | 修复 |
|------|------|
| `isTimedAuction(0)` 误判为限时拍 | `cateId<=0` 返回 false |
| 模式 B 支付回调误写号牌表 | 仅模式 A 同步 `state/paystatus` |
| 双场 session 仅一场结束可退款 | `canApplyDepositRefund` 校验 session 内全部专场 status=2 |
| 退款重复申请 | `application=1` 拦截 |
| 重新缴纳未清 application | `onPaymentSuccess` 重置 `application=0` |
| 代理跟价未校验保证金 | `automatic()` 循环内 `checkCanParticipate` |
| 编辑代理未校验 | `updateAgents` 限时拍拦截 |
| `StartSale` 仅第一场开拍 | 遍历双场 + `productList_{cateId}` |
